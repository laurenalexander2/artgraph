Art Graph

A semantic retrieval system for large-scale art-historical corpora.

The technology

Art Graph is a computational research infrastructure that represents art collections as a high-dimensional semantic space. Rather than treating objects as isolated records, the system models collections relationally, allowing meaning to emerge through proximity, clustering, and similarity across large datasets.

Institutional collection exports are ingested as versioned datasets. Each object is persisted alongside its full metadata and encoded using transformer-based language models into a shared vector space. User queries are embedded into the same space, enabling similarity-based retrieval via cosine distance rather than rigid schema constraints.

  • Backend: FastAPI with SQLite-backed persistence
  • Embedding model: all-MiniLM-L6-v2 (SentenceTransformers)
  • Similarity computation: PyTorch cosine similarity
  • Frontend: multi-page Streamlit research interface

The system prioritizes explicit data modeling, inspectable storage, and methodological extensibility. While implemented as a prototype, its architecture is designed to migrate toward ANN-backed vector databases and multimodal embeddings without altering the research paradigm.

Current limitations

The system’s outputs are shaped by the structure of institutional metadata and the representational assumptions of pretrained language models. Semantic proximity reflects shared descriptive language rather than historical influence or causal relation.

  • Results are contingent on cataloging practices and metadata quality
  • Vector similarity encodes correlation, not historical explanation
  • Image embeddings are not yet integrated
  • Similarity search operates in-memory, not via ANN (so search not that fast -- yet!)
  • I only have ~370k artworks indexed
The prototype

"Prototype" is rather.. inaccurate! Given scale/hardware constraints, more of a proof of concept. The current PoC ingests collection exports, generates and persists embeddings, and performs semantic retrieval across indexed objects. Dataset provenance, object identity, and embedding state are explicitly tracked in a relational schema.

The demonstration instance indexes open-access museum data and allows real-time semantic querying across thousands of objects without schema engineering or manual taxonomy construction.
